# Аналитический Проект: Исследование стартапов

## Цель проекта:

cоздание надежного и полного набора данных для анализа стартапов, обеспечивающего основу для эффективного инвестиционного решения путем выявления значимых признаков, повышающих вероятность успешной сделки по приобретению стартапа.


## Контекст проекта:

компания выходит на инвестиционный рынок с задачей приобретения, развития и дальнейшей продажи перспективных стартапов. Поскольку внутренние аналитики не обладают достаточным ресурсом для обработки большого объема разнородных данных, привлекаются сторонние эксперты. Первоначальные данные имеют проблемы с полнотой и качеством, что требует детальной предварительной обработки и верификации информации.

## Ключевые задачи проекта
1.	Подготовка и очистка данных
    - Объединение данных из различных источников по единым критериям;
    - Проверка качества и достоверности сведений о команде стартапа и уровне образования сотрудников;
    - Интерпретация нестандартных транзакций («нулевые» или символические суммы покупки).
2.	Аналитическое исследование данных
    - Определение влияния различных факторов (отраслевых особенностей, объемов инвестиций и стадий финансирования) на стоимость стартапа;
    - Расчет и визуализация количественных метрик и динамики изменения стоимости стартапа.
3.	Поиск индикаторов успешной сделки
    - Идентификация основных признаков и факторов, способствующих повышению вероятности удачного приобретения стартапа.

## Результат реализации проекта

разработать структурированный, качественный и валидный датасет, содержащий необходимые атрибуты и показатели, позволяющие выявить закономерности и признаки, увеличивающие шансы на выгодную покупку стартапа.

## Содержание проекта

- Знакомство с данными: загрузка и первичная предобработка:
    - Первичное знакомство с данными
    - Смена типов данных и анализ пропусков
-  Предобработка данных и их предварительное исследование:
    -  Исследование раундов финансирования по годам
    -  Изучение информации о сотрудниках и их образование
    -  Оценка целесообразности объединения данных
    -  Исследование проблем пропусков в `company_and_rounds`.
-  Исследовательский анализ финальных датасетов
    - Формирование общего датасета
    - Анализ выбросов в финальном датасете
    - Анализ аномальных продаж
    - Топ-категорий стартапов по цене и разбросу цен
    - Оценка раундов финансирования стартапа перед покупкой

- Итоговый вывод и рекомендации к проекту об исследовании стартапов

## Описание данных

Данные для исследования рынка стартапов представлены в нескольких датасетах.

Таблица `acquisition` - содержит информацию о сделке  купли - продажи компании, то есть таблица содержит данные которые необходимы будут для понимания суммах сделок, какая компания какую купила и так далее.
- `id` — идентификатор покупки;
- `acquiring_company_id` — идентификатор покупающей компании;
- `acquired_company_id` — идентификатор покупаемой компании;
- `term_code` — варианты расчёта;
- `price_amount` — сумма сделки;
- `acquired_at` — дата сделки.

Таблица `company_and_rounds` - отражает информацию о компаниях и о раундах финансирования стартапов. При этом на данную таблицу стоит обратить особое внимание в дальнешем при проведении исследвоания.

- `company ID` — идентификатор компании;
- `name` — название компании;
- `category code` — категория области деятельности компании;
- `status` — статус компании;
- `founded at` — дата инвестирования;
- `closed at` — дата закрытия компании;
- `domain` — официальный сайт компании;
- `network username` — ник компании в сети;
- `country code` — код страны компании;
- `investment rounds` — число инвестиционных раундов;
- `funding rounds` — число раундов финансирования;
- `funding total` — сумма финансирования;
- `milestones` — вехи финансирования;
- `funding round id` — идентификатор этапа финансирования;
- `company id` — идентификатор компании;
- `funded at` — дата финансирования;
- `funding round type` — тип финансирования;
- `raised amount` — сумма финансирования;
- `pre money valuation` — предварительная денежная оценка;
- `participants` — число участников;
- `is first round` — является ли раунд первым;
- `is last round` — является раунд последним.
Далее представлены 3 таблицы в которых указаны данные о работниках компаний и их образовании.

Таблица `people` хранит данные о данных о работниках представленных компаний.

- `id` — идентификатор сотрудника;
- `first_name` — имя сотрудника;
- `last_name` — фамилия сотрудника;
- `company_id` — идентификатор компании;
- `network_username` — ник в социальных сетях.

Таблица `education` включает данные оо образовании сотрудников.
- `id` — идентификатор записи об образовании;
- `person_id` — идентификатор работника;
- `instituition` — название учебного заведения;
- `graduated_at` — дата получения образования.

Таблица `degrees` - отражает данные о типе полученного образования и специальности персонала.

- `id` — идентификатор записи;
- `object_id` — идентификатор сотрудника;
- `degree_type` — тип образования;
- `subject` — специальность.
- 
Дополнительно у нас имеются данные еще 2х таблиц - `fund` и `investment`.

Таблица `fund` содержит информацию об инвестиционных фондах, которые участвуют в инвестировании и финансировании стартапов.

- `id` — идентификатор фонда;
- `name` — название фонда;
- `category_code` — категория области деятельности компании;
- `founded_at` — дата создания фонда;
- `domain` — сайт фонда;
- `network_username` — ник фонда в сети;
- `country_code` — код страны фонда;
- `investment_rounds` — число инвестиционных раундов;
- `invested_companies` — число профинансированных компаний;
- `milestones` — вехи финансирования.

Таблица `investment` - включает более подробную инфорамцию о самих раундах ивнестирования.

- `id` — идентификатор этапа инвестирования;
- `funding_round_id` — идентификатор раунда инвестирования;
- `company_id` — индентификатор компании;
- `funding_round_type` — тип финансирования;
- `fund_id` — идентификатор фонда.
